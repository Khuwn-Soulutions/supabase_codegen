# Commit Message Instructions

All commit messages should use [Conventional Commits](https://www.conventionalcommits.org/) with the following scopes applied:

## General Rules

-   **Focus on Staged Files**: When generating commit messages, focus only on the staged files unless no files are staged.

## Scopes

1.  **Core Package**:
    -   Path: `packages/supabase_codegen`
    -   Scope: `core`

2.  **Derived Packages**:
    -   Path: `packages/supabase_codegen_<suffix>`
    -   Scope: `<suffix>`
    -   Example: `packages/supabase_codegen_flutter` -> `flutter`

3.  **Examples**:
    -   If the changes are in the `example` folder of a package, append `_example` to the scope.
    -   `packages/supabase_codegen/example` -> `core_example`
    -   `packages/supabase_codegen_<suffix>/example` -> `<suffix>_example`

4.  **Other Folders**:
    -   Use the folder name with no preceding period.
    -   Example: `.github` -> `github`
